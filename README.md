# 時限管理サイトのダウンロード方法
0. このリポジトリ内のschedule-app.7zをクリックし、ページ内のView Rawを押し、ダウンロードを行う。
   ダウンロード後は以下の事項に従ってサイトを立ち上げることができる。
1. Node.jsをダウンロードしてなければURLより、ダウンロードを行う→https://nodejs.org/ja
2. 圧縮された7zファイルを解凍し、解凍したファイルをローカルディスクに置く。
3. コマンドプロンプトを立ち上げ、以下(1)(2)のコマンドを順に入力する。
   
(1) cd schedule-app 　

(2) npm run dev

5. 3の(2)のコマンドを入力した際に表示されたURLをWebブラウザにて開き、サイトに訪れれば、完了である。
6. アカウント新規登録時にユーザーIDをadminと入力すれば、これまで入力したIDとパスワードが記録された管理画面に移動できる。

# 要件定義
【目的】

主に​高校生・​大学生・​大学院生・社会人等を​主に​対象と​する。​バイトの​シフト管理・研究計画・移動計画・旅行に​おける​行動計画等の​「イベント」​全般の​管理を​行う。​公共の​場や​プライベートの​場にも​使える​

【入出力】

(登録)
イベントの​スケジュールの​発生場所(学校の​講義なら学校・旅行なら​旅行先等)、​通知設定、​ジャンル毎の​色分けスケジュールの​レスポンス設定は​必要

(出力)一日の​タイムラインを​表示・通知を​行う。​週・月単位の​タイムラインも​見れるように​する​イベントに​間に​合うように、​何時間前、​何分前、​何秒前に​通知が​来るように​する。​この​通知は​複数回行える。​最短は​15秒前～最長は​24時間前に​通知を​行えるように​する。​通知は​プッシュ通知で​行う。​

【制約】

​一つの​アドレス(アカウント)に​つき、​一つの​スケジュールのみ​管理できる。​プッシュ通知を​行う​関係上、​アプリストアから​ダウンロードできる​ネイティブアプリが​望ましいと​考える。​ただし、​場合に​よっては​webアプリでの​実装も​考慮する。​使用言語は​未定である。​

【受け入れ基準】

自分以外の​スケジュールに​アクセスできないように​する​セキュリティ要件を​満たし、​プッシュ通知が​実装されているなら、​時間の​ズレ無しに​通知が​行われるか​どうか。​スマートフォン上で、​一目で​スケジュールが​分かるような​形に​なっているかを​完了基準と​する。​

【非目標】アラーム機能はアプリ実装の都合上、見送る。有料プランは​金銭の​トラブルを​防ぐ​ため、​実装しない。​また、​プライバシーの​関係上、​webアプリまたは​ネイティブアプリ上での​他人との​スケジュールの​共有は​行えないように​する。​共有自体は​実世界上でも​可能である​ため、​現時点では​実装を​見送る。​外部​カレンダーアプリとの​連携は​実装を​見送る。
